# AI 영상편집 서비스 운영 보완 설계서

> 목적: 기존에 정리한 **기술 엔진 / 서비스 운영 페이지 / 메인페이지 / 관리자 페이지** 문서에서 빠질 수 있는 운영상 핵심 요소를 보완한다.  
> 이 문서는 Cursor AI에게 전달하여 이후 각 항목별 상세 `.md` 문서, 정책 문서, DB/API 설계, 프론트엔드 화면 설계로 확장하기 위한 상위 운영 설계서이다.

---

## 0. 문서 요약

AI 영상편집 서비스는 단순한 웹앱이 아니라, 다음 요소가 모두 연결된 **고원가·고리스크 SaaS**이다.

- 대용량 영상 업로드
- AI 분석
- 자막 생성
- 장면 분석
- 프록시 생성
- 실시간 미리보기
- 최종 렌더링
- 스토리지 보관
- 다운로드 트래픽
- 결제/구독/크레딧
- 환불/CS/저작권/초상권 대응

따라서 서비스 운영에서 가장 중요한 것은 아래 10가지이다.

```text
1. 요금제/크레딧/원가 구조
2. 온보딩/첫 성공 경험
3. 스타일 프리셋/템플릿 시스템
4. 품질 평가/AI 결과물 QA
5. CS/환불/실패 보상 정책
6. 저작권/초상권/업로드 콘텐츠 정책
7. 팀/교육기관/B2B 요금제
8. 데이터 보관/삭제 정책
9. 전환 퍼널/분석 이벤트
10. 장애 대응 런북
```

# 1. 요금제·크레딧 정책 정교화

## 1.1 왜 중요한가

AI 영상편집 서비스는 원가 구조가 복잡하다.

사용자 입장에서는 단순히 “영상 하나 편집”처럼 보이지만, 내부적으로는 다음 비용이 발생한다.

```text
영상 업로드 비용
원본 파일 저장 비용
프록시 영상 생성 비용
썸네일 추출 비용
STT 비용
장면 분석 비용
OCR/객체 탐지 비용
LLM 타임라인 생성 비용
미리보기 생성 비용
최종 렌더링 비용
다운로드 트래픽 비용
결과물 보관 비용
재렌더링 비용
실패 작업 보상 비용
```

따라서 단순히 `월 29,000원 무제한` 같은 구조로 가면, 사용량이 많은 고객에게 손해가 발생할 수 있다.

---

## 1.2 반드시 정해야 할 정책

| 항목 | 정책 결정 사항 |
|---|---|
| 무료 체험 | 몇 초/몇 분까지 무료 제공할지 |
| 무료 크레딧 | 가입 시 몇 크레딧 제공할지 |
| 크레딧 차감 기준 | 분석, 미리보기, 최종 렌더링 중 어디서 차감할지 |
| 영상 길이 제한 | 무료/유료 플랜별 최대 업로드 길이 |
| 해상도 제한 | 무료는 720p, 유료는 1080p/4K 등 |
| 저장 기간 | 무료/유료/기업별 원본·결과물 보관 기간 |
| 재렌더링 | 수정 후 재렌더링 무료 여부 |
| 실패 보상 | 렌더링 실패 시 자동 환급 여부 |
| 워터마크 | 무료 결과물에 워터마크 삽입 여부 |
| 동시 작업 수 | 플랜별 동시 렌더링 개수 |

---

## 1.3 추천 과금 구조

초기 MVP에서는 아래 구조를 추천한다.

```text
미리보기 생성: 무료 또는 매우 저렴하게 제공
최종 렌더링: 크레딧 차감
고해상도 다운로드: 크레딧 추가 차감
장기 보관: 유료 플랜 혜택
재렌더링: 일정 횟수 무료 후 차감
```

이유는 명확하다.

사용자는 결과물을 보기 전까지 결제를 망설인다. 따라서 **미리보기는 최대한 쉽게 경험하게 하고, 최종 렌더링·다운로드 단계에서 결제/크레딧을 유도**하는 것이 좋다.

---

## 1.4 추천 요금제 예시

| 플랜 | 대상 | 월 가격 예시 | 제공량 | 제한 |
|---|---|---:|---|---|
| Free | 체험 사용자 | 0원 | 짧은 미리보기, 워터마크 | 다운로드 제한, 저장 7일 |
| Starter | 개인 크리에이터 | 19,000~29,000원 | 월 60~120분 렌더링 | 1080p 제한 |
| Pro | 강사/마케터 | 49,000~99,000원 | 월 300~600분 렌더링 | 워터마크 제거 |
| Studio | 팀/소규모 회사 | 149,000~299,000원 | 팀 계정, 브랜드 템플릿 | 멤버 초대 |
| Academy | 학원/출판사 | 별도 견적 | 과목별 템플릿, 세금계산서 | 관리자 승인 |
| Enterprise | 대형 기관 | 별도 계약 | API, SLA, 전용 저장소 | 커스텀 계약 |

---

## 1.5 크레딧 차감 예시

```text
1크레딧 = 1초 1080p 최종 렌더링 기준
720p 렌더링 = 0.5크레딧/초
1080p 렌더링 = 1크레딧/초
4K 렌더링 = 3크레딧/초
STT 분석 = 0.1크레딧/초
AI 타임라인 생성 = 프로젝트당 고정 50~200크레딧
재렌더링 = 변경 범위에 따라 부분 차감
```

초기에는 너무 복잡하게 보여주지 말고 사용자에게는 아래처럼 단순화한다.

```text
이번 영상 예상 사용량: 320 크레딧
남은 크레딧: 1,240 크레딧
렌더링 후 남은 크레딧: 920 크레딧
```

---

## 1.6 관리자 페이지에 필요한 지표

```text
사용자 1명당 평균 원가
프로젝트 1개당 평균 원가
1분 영상 렌더링당 평균 원가
무료 유저 원가
유료 유저 원가
플랜별 마진율
실패 렌더링으로 낭비된 비용
GPU/CPU 사용률
스토리지 비용
트래픽 비용
```

예시 관리자 카드:

```text
오늘 매출: 320,000원
오늘 AI/렌더링 원가: 91,000원
오늘 예상 Gross Margin: 71.5%
무료 유저 원가: 38,000원
렌더링 실패 비용: 12,000원
```

# 2. 온보딩/첫 성공 경험 설계

## 2.1 왜 중요한가

AI 영상편집 서비스는 사용자가 처음 들어왔을 때 바로 결과물을 봐야 한다.

처음부터 긴 영상 업로드, 세부 설정, 복잡한 스타일 입력을 요구하면 이탈한다.

가장 중요한 지표는 단순 회원가입이 아니라 다음이다.

```text
첫 미리보기 완료율
```

즉, 사용자가 가입 후 첫 AI 편집 결과를 실제로 봤는지가 핵심이다.

---

## 2.2 추천 첫 사용 흐름

```text
회원가입
→ 예시 영상 선택 또는 직접 업로드
→ 제목/내용/스타일 입력
→ AI가 30~60초 샘플 편집
→ 실시간 미리보기
→ “이 정도면 쓸 만하다” 경험
→ 최종 렌더링/다운로드/결제 유도
```

---

## 2.3 메인 CTA 문구 방향

약한 CTA:

```text
무료로 시작하기
```

더 강한 CTA:

```text
영상 하나 넣고 AI 편집본 바로 보기
1분 만에 내 영상이 쇼츠로 바뀌는지 확인하기
원본 영상 업로드하고 자동 편집 미리보기
```

---

## 2.4 온보딩 화면 구성

| 단계 | 화면 | 목적 |
|---|---|---|
| Step 1 | 사용 목적 선택 | 쇼츠/강의/광고/인터뷰/브이로그 구분 |
| Step 2 | 영상 업로드 또는 샘플 선택 | 사용 장벽 낮추기 |
| Step 3 | 제목/내용 입력 | AI 편집 방향 설정 |
| Step 4 | 스타일 선택 | 결과물 기대치 형성 |
| Step 5 | AI 편집 미리보기 | 첫 성공 경험 제공 |
| Step 6 | 수정 명령/다운로드 | 결제 전환 유도 |

---

## 2.5 샘플 프로젝트 필요성

사용자가 영상 파일을 준비하지 않아도 서비스를 체험할 수 있어야 한다.

샘플 프로젝트 예시:

```text
강의 영상 → 해설 쇼츠
인터뷰 영상 → 핵심 요약 쇼츠
제품 소개 영상 → 광고 영상
브이로그 영상 → 하이라이트 영상
문제 이미지 + 풀이 → 교육 쇼츠
```

# 3. 템플릿·스타일 프리셋 시스템

## 3.1 왜 중요한가

사용자가 “원하는 편집 스타일”을 자유롭게 입력하면 결과 품질이 들쑥날쑥해질 수 있다.

따라서 내부적으로는 사용자의 자연어 입력을 **정규화된 스타일 프리셋**으로 변환해야 한다.

---

## 3.2 기본 스타일 프리셋 예시

| 스타일 | 용도 | 특징 |
|---|---|---|
| 빠른 쇼츠형 | 릴스/틱톡/쇼츠 | 빠른 컷, 큰 자막, 강한 강조 |
| 강의 해설형 | 학원/교육 | 문제 확대, 풀이 단계, 깔끔한 자막 |
| 인터뷰 요약형 | 인터뷰/팟캐스트 | 발화자 중심, 핵심 문장 강조 |
| 제품 소개형 | 스타트업/사업자 | 기능 소개, CTA, 브랜드 컬러 |
| 브이로그 컷편집형 | 일상/여행 | 감성 BGM, 자연스러운 컷 |
| 뉴스 요약형 | 정보성 콘텐츠 | 헤드라인, 요약 카드, 빠른 정보 전달 |
| 토스형 미니멀 | 깔끔한 브랜드형 | 흰 배경, 큰 타이포, 파란 포인트 |

---

## 3.3 자연어 → 프리셋 변환 예시

사용자 입력:

```text
토스처럼 깔끔하고 빠르게 편집해줘. 자막은 크게 넣고 너무 정신없지는 않게 해줘.
```

내부 변환:

```json
{
  "preset": "minimal_clean",
  "captionSize": "large",
  "motion": "soft",
  "transition": "cut",
  "colorAccent": "blue",
  "bgmIntensity": "low",
  "pace": "medium_fast",
  "visualDensity": "low"
}
```

---

# 4. 품질 평가/QA 시스템

## 4.1 왜 중요한가

AI가 자동 편집한 결과물이 좋은지 판단하는 내부 기준이 필요하다.

단순히 작업 성공/실패만 보면 안 된다. 성공했더라도 품질이 낮으면 사용자는 결제하지 않는다.

---

## 4.2 평가 항목

| 평가 항목 | 설명 |
|---|---|
| 자막 정확도 | STT 오타율, 문장 분리 품질 |
| 컷 편집 품질 | 무음 구간 제거, 불필요 구간 제거 |
| 템포 | 너무 빠르거나 느리지 않은지 |
| 화면 구성 | 얼굴/문제/자막이 잘리지 않았는지 |
| 싱크 | 자막과 음성이 맞는지 |
| 렌더링 오류 | 검은 화면, 깨짐, 소리 누락 여부 |
| 사용자 수정 횟수 | 수정이 많을수록 최초 품질 낮음 |
| 다운로드 전환율 | 미리보기 후 다운로드했는지 |
| 재사용률 | 사용자가 다시 프로젝트를 만들었는지 |

---

## 4.3 품질 점수 구조

```json
{
  "projectId": "proj_123",
  "qualityScore": 84,
  "captionAccuracy": 91,
  "cutQuality": 78,
  "visualSafety": 88,
  "audioSync": 95,
  "renderIntegrity": 100,
  "userRevisionCount": 2,
  "downloaded": true
}
```

---

## 4.4 관리자 페이지에 필요한 QA 화면

```text
프로젝트 목록
→ 품질 점수 낮은 순 정렬
→ 결과 영상 미리보기
→ 사용된 프롬프트/모델 버전 확인
→ 사용자 수정 명령 로그 확인
→ 실패 원인 태깅
→ 좋은 결과물/나쁜 결과물 분류
```

# 5. 비용 모니터링·원가 관리

## 5.1 왜 중요한가

AI 영상편집은 원가 폭탄이 날 수 있다.

특히 무료 사용자에게 많은 리소스를 열어두면 매출보다 비용이 먼저 커질 수 있다.

---

## 5.2 반드시 추적할 원가 항목

```text
STT 비용
LLM 비용
영상 분석 비용
렌더링 CPU/GPU 비용
프록시 생성 비용
스토리지 비용
CDN/다운로드 트래픽 비용
실패 작업 재시도 비용
무료 사용자당 평균 원가
유료 사용자당 평균 원가
플랜별 마진율
```

---

## 5.3 원가 데이터 구조 예시

```ts
type ProjectCost = {

```

# 6. 저작권·초상권·콘텐츠 정책

## 6.1 왜 중요한가

영상 서비스는 법적 리스크가 크다.

사용자가 업로드하는 콘텐츠에는 다음이 포함될 수 있다.

```text
타인의 얼굴
강의 영상
유튜브 영상 다운로드본
방송 클립
음악
저작권 이미지
학생 얼굴
수업 자료
문제집/PDF
시험지
출판사 교재
```

노바AI/교육용 영상 서비스라면 특히 **문제집·시험지·강의자료 저작권** 이슈가 중요하다.

---

## 6.2 필요한 정책

| 정책 | 설명 |
|---|---|
| 업로드 콘텐츠 권리 고지 | 사용자가 권리를 가진 콘텐츠만 업로드해야 함 |
| 초상권 책임 고지 | 타인 얼굴 포함 시 사용자가 동의 확보 |
| 저작권 신고 절차 | 권리자가 신고할 수 있는 창구 |
| 신고 콘텐츠 처리 | 신고 시 비공개/삭제/검토 프로세스 |
| 교육자료 사용 정책 | 문제집/PDF/시험지 업로드 책임 범위 |
| AI 생성물 책임 범위 | 결과물 검수 책임은 사용자에게 있음을 고지 |
| 미성년자/학생 얼굴 정책 | 학생 영상 업로드 시 동의 필요 |
| 보관/삭제 정책 | 사용자의 원본·결과물 삭제 요청 처리 |

---

## 6.3 회원가입/업로드 시 고지 문구 예시

```text
업로드하는 영상, 이미지, 음원, 문서에 대한 사용 권한이 본인에게 있음을 확인합니다. 타인의 얼굴, 음성, 저작물, 교육자료가 포함된 경우 필요한 동의와 권리를 확보해야 하며, 이와 관련한 법적 책임은 업로드한 사용자에게 있습니다.
```

---

## 6.4 관리자 페이지에 필요한 기능

```text
신고 콘텐츠 목록
사용자별 업로드 기록
문제 콘텐츠 비공개 처리
프로젝트 강제 삭제
권리자 신고 메모
정책 위반 사용자 정지
반복 위반자 제한
```

# 7. 결제 이후 고객 성공 흐름

## 7.1 왜 중요한가

많은 SaaS는 결제는 받지만 사용자가 제대로 활용하지 못해 이탈한다.

AI 영상편집 서비스는 결과 생성 시간이 있고, 사용자가 “어떻게 써야 할지” 모르면 재사용률이 낮다.

---

## 7.2 결제 직후 흐름

```text
결제 완료
→ 플랜 혜택 요약
→ 첫 프로젝트 만들기 버튼
→ 추천 템플릿 제안
→ 사용량/크레딧 표시
→ 튜토리얼 영상 또는 샘플 프로젝트 제공
→ 첫 결과물 생성 완료 알림
```

---

## 7.3 필수 알림

```text
결제 완료 알림
크레딧 충전 알림
렌더링 시작 알림
렌더링 완료 알림
렌더링 실패 알림
크레딧 부족 알림
저장 기간 만료 예정 알림
구독 갱신 예정 알림
구독 결제 실패 알림
미사용 고객 리텐션 알림
```

---

## 7.4 알림 채널

| 채널 | 용도 |
|---|---|
| 이메일 | 결제, 결과물, 정산, 정책 안내 |
| 카카오 알림톡 | 렌더링 완료, 결제 실패, 중요 알림 |
| 사이트 내 알림 | 작업 상태, 크레딧, 시스템 공지 |
| 브라우저 푸시 | 렌더링 완료 알림 |
| 문자 | B2B/고액 고객 긴급 안내 |

---

## 8.2 CS 처리 정책 예시

|---|---|

| 파일 삭제 | 보관 기간 지난 경우 복구 불가 고지 |
| 업로드 실패 | 파일 포맷/용량 확인 후 재업로드 안내 |
| 다운로드 실패 | 다운로드 링크 재생성 |
| 스타일 미반영 | 프리셋 재적용 또는 수정 명령 안내 |

# 9. 권한·팀 기능

## 9.1 왜 중요한가

B2C만 보면 팀 기능이 필요 없어 보이지만, 학원/회사 고객을 생각하면 매우 중요하다.

예를 들어 학원 원장이 결제하고, 선생님과 조교가 함께 사용하는 구조가 많을 수 있다.

---

## 9.2 추천 권한 구조

| 권한 | 설명 |
|---|---|
| Owner | 결제, 멤버 관리, 전체 프로젝트 관리 |
| Admin | 팀 프로젝트 관리, 템플릿 관리, 사용량 확인 |
| Editor | 영상 생성/편집/다운로드 가능 |
| Viewer | 결과물 확인만 가능 |
| Billing Manager | 결제/세금계산서/영수증 관리 가능 |

---

## 9.3 학원 계정 예시

```text
대치 ○○학원 계정
- 원장님: 결제 및 멤버 관리
- 물리 선생님: 물리 해설 쇼츠 생성
- 수학 선생님: 수학 문제풀이 영상 생성
- 조교: 업로드/다운로드 담당
- 마케팅 담당자: 릴스/쇼츠 게시용 영상 다운로드
```

---

## 9.4 팀 기능 MVP 범위

초기에는 너무 복잡한 조직 관리까지 만들 필요는 없다.

MVP 권장 범위:

```text
팀 생성
멤버 초대
권한 설정
팀 크레딧 공유
프로젝트 공유
결제 담당자 지정
```

---

# 10. B2B·교육기관 요금제

## 10.1 왜 중요한가

일반 개인 요금제만 있으면 객단가가 낮다.

노바AI의 기존 고객층인 학원 선생님, 출판사, 교육기관을 고려하면 B2B 요금제를 반드시 설계해야 한다.

---

## 10.2 B2B 요금제 예시

| 플랜 | 대상 | 주요 기능 |
|---|---|---|
| Academy Basic | 소규모 학원 | 팀 계정, 월간 공유 크레딧 |
| Academy Pro | 중대형 학원 | 과목별 템플릿, 로고 워터마크 |
| Publisher | 출판사 | 대량 영상 생성, 검수 워크플로우 |
| Enterprise | 대형 교육기관 | 전용 서버, API, SLA, 계약서 기반 정산 |

---

## 10.3 교육기관 전용 기능

```text
팀 멤버 초대
월간 영상 생성량 공유
브랜드 템플릿 저장
학원 로고 워터마크
과목별 템플릿
관리자 승인 후 다운로드
세금계산서/견적서 발행
대량 프로젝트 생성
과목별 폴더 관리
강사별 사용량 리포트
```

---

## 10.4 B2B 영업용 페이지 필요

메인페이지와 별도로 `/academy` 또는 `/education` 랜딩페이지를 만드는 것이 좋다.

카피 방향:

```text
문제와 강의 영상을 넣으면 AI가 해설 쇼츠를 만들어줍니다.
학원 홍보용 릴스, 개념 설명 영상, 문제풀이 쇼츠를 자동 생성하세요.
```

# 11. 데이터 보관·삭제 정책

## 11.1 왜 중요한가

영상은 용량이 크다. 원본 파일과 렌더링 결과물을 무제한 보관하면 스토리지 비용이 계속 증가한다.

따라서 플랜별 보관 정책이 반드시 필요하다.

---

## 11.2 추천 보관 정책

| 사용자 유형 | 원본 파일 | 프록시 파일 | 최종 결과물 | 프로젝트 데이터 |
|---|---:|---:|---:|---:|
| 무료 사용자 | 3일 | 7일 | 7일 | 30일 |
| Starter | 7일 | 14일 | 30일 | 90일 |
| Pro | 30일 | 30일 | 90일 | 180일 |
| Studio | 60일 | 60일 | 180일 | 1년 |
| Enterprise | 계약 조건 | 계약 조건 | 계약 조건 | 계약 조건 |

---

## 11.3 사용자 고지 문구 예시

```text
무료 플랜의 원본 파일은 업로드 후 3일 동안 보관되며, 최종 결과물은 7일 후 자동 삭제됩니다. 장기 보관이 필요한 경우 유료 플랜을 이용해주세요.
```

---

## 11.4 시스템 작업

```text
매일 만료 파일 스캔
삭제 예정 파일 알림
만료된 원본 파일 삭제
만료된 프록시 파일 삭제
만료된 결과물 삭제
DB에는 삭제 상태 기록
관리자 삭제 로그 저장
```

# 12. 온사이트 분석·전환 퍼널

# 13. 프롬프트·모델 버전 관리

```text
프롬프트별 다운로드율 비교
프롬프트별 평균 수정 횟수 비교
```

# 14. 장애 대응·운영 런북

특히 대용량 파일, 외부 AI API, GPU/CPU 작업, 결제 웹훅, 스토리지 권한 등에서 오류가 자주 발생할 수 있다.

---

## 14.2 필요한 런북

```text

GPU 서버 다운 대응
결제 웹훅 실패 대응
저장소 권한 오류 대응
다운로드 링크 만료 대응
크레딧 차감 오류 대응
프록시 생성 실패 대응
외부 모델 API 장애 대응
```

---

## 14.3 관리자 액션

```text
작업 재시도

작업 취소
작업 우선순위 변경
사용자에게 공지
```

# 15. 메인페이지 이후 랜딩 페이지 분화

## 15.1 왜 중요한가

메인페이지 하나로 모든 고객을 설득하기 어렵다.

크리에이터, 강사, 사업자, 학원 원장은 각각 다른 페인포인트를 가진다.

---

## 15.2 추천 랜딩 페이지

| URL | 타깃 | 메시지 |
|---|---|---|
| `/ai-shorts` | 크리에이터/마케터 | 긴 영상을 AI가 쇼츠로 자동 편집 |
| `/ai-lecture-video` | 학원 선생님/강사 | 문제와 강의 영상을 해설 쇼츠로 자동 변환 |
| `/ai-product-video` | 사업자/스타트업 | 제품 설명만 넣으면 광고 영상 자동 생성 |
| `/ai-interview` | 팟캐스트/인터뷰 | 긴 인터뷰에서 핵심 클립 자동 추출 |
| `/academy` | 학원/B2B | 학원 홍보·해설 영상 제작 자동화 |

# 16. Cursor AI 개발 지시서

## 16.1 전체 목표

Cursor AI는 이 문서를 기반으로 AI 영상편집 SaaS 운영 시스템을 설계해야 한다.

중점은 다음이다.

```text
기능을 많이 만드는 것보다,
```

# 17. 최종 결론

현재까지 만든 문서가 다음에 집중했다면,

```text
기술 엔진
서비스 운영 페이지
메인페이지 디자인
관리자 페이지
```

이번 보완 문서는 다음을 채운다.

```text
돈을 벌면서 망하지 않기 위한 운영 구조
고객이 첫 성공 경험을 하도록 만드는 온보딩
AI 결과물 품질을 관리하는 QA 시스템
원가 폭탄을 막는 크레딧/비용 관리
환불/CS/저작권/장애 대응 정책
B2B 확장을 위한 팀/교육기관 요금제
```

가장 먼저 깊게 설계해야 하는 것은 다음 3개다.

```text
1. 요금제·크레딧·원가 정책
2. 온보딩·첫 미리보기 완료 경험
3. 관리자 CS·렌더링 실패·환급 처리 흐름
```

이 3개가 약하면 서비스가 만들어져도 실제 운영에서 문제가 생긴다.

반대로 이 3개가 잘 잡히면, AI 영상편집 기술이 완벽하지 않아도 초기 고객에게 충분히 판매하고 개선할 수 있다.
